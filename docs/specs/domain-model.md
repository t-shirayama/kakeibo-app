# ドメインモデル定義

## 方針

本プロジェクトはドメイン駆動設計（DDD）を採用する。

家計簿アプリの中心はUIやデータベースではなく、家計管理に関する業務ルールである。支出、収入、明細、カテゴリ、アップロード、集計、レポートなどの概念をドメインモデルとして表現し、アプリケーションの振る舞いはドメインを中心に組み立てる。

## 横断業務ルール

- 金額は日本円を前提にし、整数JPYとして扱う。
- 一覧、集計、レポートはログインユーザー本人のデータのみを扱う。
- PDFアップロード、明細登録、明細編集、カテゴリ変更などの変更操作は、成功・失敗がユーザーに分かるようにする。
- 明細、カテゴリ、アップロード履歴は論理削除する。
- 初期カテゴリはユーザー作成時にユーザーごとに複製する。
- 使用中カテゴリは物理削除せず、既存明細のカテゴリ参照を維持したまま論理削除または無効化する。
- 無効化または論理削除されたカテゴリに紐づく明細は、明細一覧の表示と未分類絞り込みでは未分類として扱う。
- 明細編集、明細削除、アップロード失敗などの重要操作は監査ログとして残す。
- メモの最大文字数は500文字とする。

## 集約

### 明細

ユーザーの支出または収入の1取引を表す。

- 識別子: 明細ID
- 所有者: ユーザーID
- 値: 日付、店名、カテゴリ、金額、支払い方法、利用者、メモ、種別
- 不変条件:
  - 日付は必須
  - 店名は必須
  - カテゴリは必須
  - 金額は整数JPY
  - 金額がマイナスの場合は取消明細を表す
  - 0円明細を許可する
  - 種別は支出または収入

収入明細は支出明細と同じ `transactions` テーブルで扱い、`transaction_type` で区別する。

### カテゴリ

明細を分類するための項目を表す。

- 識別子: カテゴリID
- 所有者: ユーザーID
- 値: カテゴリ名、色、説明、月次予算、有効状態
- 不変条件:
  - カテゴリ名は必須
  - 同一ユーザー内でカテゴリ名は重複しない
  - 月次予算は未設定または0以上の整数JPY
  - 使用中カテゴリは物理削除しない

カテゴリの月次予算は、そのカテゴリに属する支出明細の当月実績と比較するために使う。収入カテゴリに予算を持たせてもよいが、ダッシュボードの予算進捗は支出明細だけを対象に集計する。

### 収入設定

家族や本人ごとの毎月収入、登録期間、月別変更を表す。

- 識別子: 収入設定ID
- 所有者: ユーザーID
- 値: 対象者名、収入カテゴリ、毎月の金額、毎月の発生日、登録開始月、登録終了月、月別変更
- 不変条件:
  - 対象者名は必須
  - 収入カテゴリは必須
  - 金額は0以上の整数JPY
  - 発生日は1日から31日の範囲
  - 登録開始月は必須
  - 登録終了月は未設定または登録開始月以降
  - 同じ収入設定・同じ対象月の月別変更は1件のみ

収入設定は登録開始月から登録終了月までの各月を対象にし、発生日を迎えた月は同じ収入設定・同じ対象月で重複しないよう収入明細を自動追加する。対象月に月別変更がある場合は、基本設定ではなく月別変更の金額と発生日を使う。

発生日が月末日を超える月は、その月の末日を発生日として扱う。

### アップロード履歴

PDF明細のアップロードと解析結果を表す。

- 識別子: アップロードID
- 所有者: ユーザーID
- 値: ファイル名、アップロード日時、取込件数、ステータス、エラー理由
- 不変条件:
  - ファイル名は必須
  - ステータスは処理中、完了、失敗のいずれか
  - 取込件数は0以上

### 監査ログ

明細編集、明細削除、アップロード失敗などの重要な操作や失敗を記録する。

- 識別子: 監査ログID
- 所有者: ユーザーID
- 値: 操作種別、対象リソース種別、対象リソースID、詳細、作成日時

## 値オブジェクト

- 金額: 整数JPY。マイナスの場合は取消明細を表し、0円も許可する
- 日付: `Asia/Tokyo` を基準にした日付
- 日時: DBに保存する日時
- 店名: 明細の取引先名
- カテゴリ名: ユーザーが識別できる分類名
- 支払い方法: クレジットカード、Suica、現金など
- 利用者: カード明細上の利用者
- 対象者名: 収入設定で家族や本人を識別する名前
- ファイル名: アップロードされたPDFファイル名

## ドメインサービス候補

- PDF明細の重複判定
- 店名やメモに基づくカテゴリ推定
- 楽天カード明細PDFのルールベース抽出
- 過去分類に基づくカテゴリ自動分類
- 収入設定に基づく月次収入明細の自動追加
- 月次集計
- カテゴリ別集計
- 予算進捗集計
