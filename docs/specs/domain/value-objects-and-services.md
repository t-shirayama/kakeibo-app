# 値オブジェクトとドメインサービス候補

## 値オブジェクト

- 金額: 整数JPY。マイナスの場合は取消明細を表し、0円も許可する
- 日付: `Asia/Tokyo` を基準にした日付
- 日時: DBに保存する日時
- 店名: 明細の取引先名
- カテゴリ名: ユーザーが識別できる分類名
- 支払い方法: クレジットカード、Suica、現金など
- 利用者: カード明細上の利用者
- 対象者名: 収入設定で家族や本人を識別する名前
- ファイル名: アップロードされたPDFファイル名

## ドメインサービス候補

- PDF明細の重複判定
- 店名やメモに基づくカテゴリ推定
- 楽天カード明細PDFのルールベース抽出
- 過去分類に基づくカテゴリ自動分類
- 収入設定に基づく月次収入明細の自動追加
- 月次集計
- カテゴリ別集計
- 予算進捗集計
