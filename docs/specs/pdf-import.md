# PDF取込仕様

## 方針

初期対応は楽天カード明細PDFを対象とする。

今後、カード会社や明細フォーマットを少しずつ追加できるように、PDF解析はフォーマットごとの抽出ルールとして拡張できる設計にする。

## 初期対象

- 楽天カード明細PDF

## 抽出方式

アップロードされた明細PDFは、テーブルの列が固定されている前提でルールベース抽出する。

想定列:

- 利用日
- 利用店名
- 利用者
- 支払方法
- 利用金額

## 取込データへのマッピング案

| PDF列 | 明細項目 |
| --- | --- |
| 利用日 | 日付 |
| 利用店名 | 店名 |
| 支払方法 | 支払い方法 |
| 利用金額 | 金額 |
| 利用者 | 利用者 |

カテゴリは店名やメモに基づいて推定する。推定できない場合は未分類にする。

支払い方法は楽天PDFの値をそのまま保存する。

利用金額がマイナスの場合は取消明細として取り込む。

アップロード済みPDFは原本として保存する。

初期のPDF原本保存先はローカルの `storage/uploads/` とする。将来はS3互換ストレージへの移行を想定する。

PDF原本はユーザーが削除するまで保存する。

重複判定キーは以下を基本とする。

- ユーザーID
- 利用日
- 利用店名
- 利用者
- 支払方法
- 利用金額
- `source_upload_id` 以外の抽出元情報

## 拡張方針

- PDFフォーマットごとに抽出ルールを分ける。
- 解析結果はドメイン明細へ直接保存せず、アプリケーション層で検証してから登録する。
- 抽出できなかった行は失敗理由とともに確認できるようにする。
- 重複明細の取込を防ぐ。

## 確認事項

- 楽天カード明細PDFのサンプル行をどの形式でテストデータ化するか。
- `source_upload_id` 以外の抽出元情報として何を保存するか。
