# PDF取込仕様

## 方針

初期対応は楽天カード明細PDFを対象とする。

今後、カード会社や明細フォーマットを少しずつ追加できるように、PDF解析はフォーマットごとの抽出ルールとして拡張できる設計にする。

## 初期対象

- 楽天カード明細PDF

## 抽出方式

アップロードされた明細PDFは、テーブルの列が固定されている前提でルールベース抽出する。

想定列:

- 利用日
- 利用店名
- 利用者
- 支払方法
- 利用金額

## 取込データへのマッピング案

| PDF列 | 明細項目 |
| --- | --- |
| 利用日 | 日付 |
| 利用店名 | 店名 |
| 支払方法 | 支払い方法 |
| 利用金額 | 金額 |
| 利用者 | 利用者 |

カテゴリは店名やメモに基づいて推定する。推定できない場合は未分類にする。

アップロード済みPDFは原本として保存する。

## 拡張方針

- PDFフォーマットごとに抽出ルールを分ける。
- 解析結果はドメイン明細へ直接保存せず、アプリケーション層で検証してから登録する。
- 抽出できなかった行は失敗理由とともに確認できるようにする。
- 重複明細の取込を防ぐ。

## 確認事項

- 楽天カード明細PDFのサンプル行をどの形式でテストデータ化するか。
- 支払方法はPDFの値をそのまま保存するか、アプリ内の支払い方法へ正規化するか。
- 利用金額がマイナスや取消行を含む場合の扱い。
- 重複判定に使う項目。
- PDF原本の保存先と保存期間。
