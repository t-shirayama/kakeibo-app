# セキュリティ仕様

## 認証

認証はJWTを使う。

## 認可

- 一覧、集計、レポート、設定、アップロード履歴はログインユーザー本人のデータのみを扱う。
- APIではリクエストされたリソースの所有者を必ず検証する。

## パスワード

- パスワードはハッシュ化して保存する。
- 平文パスワードをDBやログに保存しない。

## JWT

未決定事項:

- アクセストークンの有効期限
- リフレッシュトークンを使うか
- JWTの保存場所
- ログアウト時のトークン失効方式

## ファイルアップロード

- PDFファイルのみを受け付ける。
- ファイルサイズ上限を設ける。
- 解析できないPDFは失敗として履歴に残す。
- アップロードされたファイルの保存先と保存期間は未決定。

- JWTはHttpOnly Cookieに保存するか。
- CSRF対策をどうするか。
- 最大アップロードサイズを何MBにするか。
- アップロード済みPDFを保存するか、解析後に破棄するか。
