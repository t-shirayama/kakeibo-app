# 未完了タスク

このファイルは未完了の実装タスクだけを管理します。

完了済みタスクは [完了済みタスク](completed.md) に退避しています。

仕様の正本は `docs/specs/` 配下です。タスク実行中に仕様変更が発生した場合は、関連するSSOT文書を同じ作業内で更新してください。

## 42. 明細一覧のページサイズを設定値と仕様に合わせて統一する

- [ ] 明細一覧APIとフロントエンドの `page_size` 初期値を、仕様どおり `10` 件へ統一する。
- [ ] ユーザー設定の `page_size` を明細一覧の初期表示件数へ反映し、`10` / `20` / `50` 以外を選べないようにする。
- [ ] API、画面要件、E2E、単体テストを更新し、設定変更後の表示件数が維持されることを確認する。

## 43. OpenAPIベースのAPIクライアント自動生成へ移行する

- [ ] 現在の手書きAPIクライアントを棚卸しし、OpenAPIから自動生成する構成へ置き換える。
- [ ] 型定義、APIエラーDTO、ページネーションDTOを生成物ベースで利用できるようにする。
- [ ] 生成手順をCIと開発ドキュメントへ反映し、仕様変更時にクライアントとの差分が残りにくい運用へ寄せる。

## 44. 監査ログの参照機能を追加する

- [ ] 監査ログの一覧取得APIを追加し、日時、操作種別、対象、実行ユーザーで追えるようにする。
- [ ] 設定画面または管理者向け画面から監査ログを参照できるUIを追加する。
- [ ] 明細編集、削除、アップロード失敗に加えて、重要操作が不足なく記録・表示されることをテストで確認する。

## 45. PDFアップロード処理の進捗表示と再試行導線を追加する

- [ ] PDFアップロード中の状態、完了、失敗理由を画面上で分かりやすく表示する。
- [ ] 失敗したアップロードに対して再試行しやすい導線を追加する。
- [ ] アップロード履歴、監査ログ、E2Eを更新し、大きめのPDFや失敗ケースでも状態が追えることを確認する。

## 46. 画面状態をURLクエリへ同期して共有しやすくする

- [ ] 明細一覧の検索語、カテゴリ、期間、ページング、ソート状態をURLクエリへ同期する。
- [ ] ダッシュボードとカレンダーの表示年月もURLで復元できるようにする。
- [ ] リロードや共有URLアクセス後も同じ状態が再現されることをE2Eで確認する。
