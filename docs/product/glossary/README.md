# 用語集

## 家計簿ドメイン

| 用語 | 意味 |
| --- | --- |
| 明細 | 支出または収入の1取引 |
| 支出 | ユーザーのお金が出ていく取引 |
| 収入 | ユーザーのお金が入ってくる取引 |
| 収入設定 | 家族や本人ごとの毎月収入と発生日を管理する設定 |
| 月別変更 | 収入設定の金額または発生日を特定月だけ上書きする設定 |
| カテゴリ | 明細を分類する項目 |
| 未分類 | カテゴリが確定していない明細の分類 |
| 支払い方法 | クレジットカード、Suica、現金などの決済手段 |
| メモ | ユーザーが明細に付与する補足情報 |
| アップロード | PDF明細をシステムへ送信する操作 |
| アップロード履歴 | PDFアップロードと解析結果の記録 |
| 取込 | PDFから抽出した明細を家計簿データとして登録すること |
| レポート | 支出傾向を集計・可視化したもの |
| 前月比 | 対象月と前月の値を比較した差分 |

## 技術用語

| 用語 | 意味 |
| --- | --- |
| DDD | ドメイン駆動設計 |
| 集約 | 一貫性を守るドメインオブジェクトのまとまり |
| エンティティ | 識別子で同一性を判定するドメインオブジェクト |
| 値オブジェクト | 値そのもので等価性を判定するドメインオブジェクト |
| ユースケース | ユーザー操作や業務処理を実現するアプリケーション層の処理 |
| DTO | レイヤ間やAPI境界でデータを受け渡すためのオブジェクト |
| SSOT | Single Source of Truth の略。仕様や設計について、唯一の正しい情報源として扱う正本 |
| ADR | Architecture Decision Record の略。重要なアーキテクチャ上の決定を、背景・決定内容・結果とともに記録する文書 |
| ADRs | ADRの複数形。`docs/architecture/adrs/` 配下に配置する |
| FastAPI | バックエンドAPI実装に使うPython Webフレームワーク |
| Next.js | フロントエンド実装に使うReactフレームワーク |
