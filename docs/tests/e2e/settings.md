# 設定E2Eシナリオ

## 対応テストファイル

- `frontend/e2e/settings.spec.ts`

## 目的

設定値の表示、保存、全データ削除UIのガードを確認する。

## 前提データ

- 認証済みstorage stateを使う。
- サンプルユーザー設定として表示通貨 `JPY`、1ページあたり件数 `10`、日付形式 `yyyy/MM/dd` が投入されている。

## 操作手順

- `/settings` を開く。
- 初期設定値を確認する。
- 1ページあたり件数、日付形式を変更する。
- 保存ボタンを押す。
- 全データ削除セクションを確認する。

## 期待結果

- 初期設定値が表示される。
- 保存後に `保存しました。` が表示される。
- 削除確認文字列の入力欄が表示される。
- 確認文字列なしでは `全データを削除` ボタンが無効である。

## 安定化メモ

- 実際の全データ削除は実行しない。
- 設定変更はE2E専用DB上で行い、次回実行前のDBリセットで初期化する。
