# 収入設定E2Eシナリオ

## 対応テストファイル

- `frontend/e2e/income-settings.spec.ts`

## 目的

収入設定画面で、家族ごとの毎月収入を追加し、登録期間と対象月だけの金額・発生日変更を設定できることを確認する。

## 前提データ

- 認証済みstorage stateを使う。
- 収入カテゴリとして既存カテゴリを選択できる。
- 追加する対象者名は `E2E収入` とする。

## 操作手順

- `/income-settings` を開く。
- 対象者名、毎月の金額、発生日、登録開始月、登録終了月、カテゴリを入力して収入設定を追加する。
- 追加した収入設定に月別変更を保存する。
- 追加した収入設定を削除する。

## 期待結果

- 収入設定画面が表示される。
- 追加した収入設定が一覧に表示される。
- 一覧に登録期間が表示される。
- 月別変更の金額と発生日が一覧に表示される。
- 削除後、追加した収入設定は表示されない。

## 安定化メモ

- 作成したE2E収入設定は同じテスト内で削除する。
- 明細自動追加の重複防止はAPIテストで検証し、E2Eでは画面操作とAPI接続を確認する。
