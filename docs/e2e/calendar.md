# カレンダーE2Eシナリオ

## 対応テストファイル

- `frontend/e2e/calendar.spec.ts`

## 目的

カレンダー画面で日別支出、月間サマリー、カテゴリ別サマリー、選択日の明細確認導線が表示されることを確認する。

## 前提データ

- 認証済みstorage stateを使う。
- サンプルデータには `2026-05-01` の支出明細として `成城石井` が含まれる。
- サンプルカテゴリとして `食費` が含まれる。

## 操作手順

- `/calendar` を開く。
- カレンダー見出し、表示月、月間サマリー、カテゴリ別サマリーを確認する。
- `2026-05-01` の日付セルを選択する。
- 選択日の明細に `成城石井` が表示されることを確認する。
- `明細一覧で確認` を押す。

## 期待結果

- `カレンダー` 見出しと `日別の支出カレンダー` が表示される。
- 月間サマリーに `収入`、`支出`、`収支` が表示される。
- カテゴリ別サマリーに `食費` が表示される。
- 選択日の明細一覧に `成城石井` が表示される。
- 明細一覧へ遷移すると `date_from=2026-05-01` と `date_to=2026-05-01` がURLに反映される。

## 安定化メモ

- 日付セルは `aria-label` に日付と金額を持たせ、E2Eから安定して選択できるようにする。
- 金額の厳密値はAPIテスト側に寄せ、E2Eでは代表データと画面導線を中心に確認する。
