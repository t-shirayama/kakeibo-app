# ダッシュボードE2Eシナリオ

## 対応テストファイル

- `frontend/e2e/dashboard.spec.ts`

## 目的

ダッシュボードで当月指標、予算進捗、表示月切り替え、カテゴリ別支出割合、収入・支出推移、最近の明細が表示されることを確認する。

## 前提データ

- 認証済みstorage stateを使う。
- サンプルデータには当月の明細として `成城石井` が含まれる。
- サンプルカテゴリとして `食費` が含まれる。

## 操作手順

- `/dashboard` を開く。
- ダッシュボード見出しを確認する。
- 指標、予算進捗、カテゴリ別支出割合の円グラフ、凡例フォーカス時のハイライト、収入・支出推移、最近の明細の各領域を確認する。
- 表示月の入力が当月で初期表示されることを確認する。
- 前月ボタンを押し、表示月とKPI見出しが前月に切り替わることを確認する。
- 翌月ボタンを押し、当月表示に戻ることを確認する。
- カテゴリ別支出割合の `食費` をクリックする。

## 期待結果

- `今月の支出合計`、`今月の収入合計`、`今月の残高`、`取引件数` が表示される。
- 予算進捗として、予算内か超過か、進捗率、カテゴリ別予算進捗が表示される。
- 表示月は `YYYY-MM` 形式で表示され、前月・翌月ボタンに応じてダッシュボード集計APIの `year` / `month` 指定が切り替わる。
- カテゴリ別支出割合の円グラフが表示される。
- カテゴリ別支出割合に `食費` が表示される。
- カテゴリ別支出割合の凡例は金額の大きい順に表示される。
- カテゴリ別支出割合の凡例にフォーカスすると対応する円グラフ部分がハイライト表示される。
- カテゴリ別支出割合の凡例がスクロールする場合でも、画面全体のスクロールへ波及しない。
- `食費` をクリックすると、明細一覧へ遷移し、表示月の `date_from` / `date_to` と選択カテゴリの `category_id` がURLに反映される。
- 遷移先の明細一覧で、対象月・対象カテゴリの明細として `成城石井` が表示される。
- 直近6ヶ月の月別収入支出グラフと、収入・支出・平均支出の凡例、平均支出ラインが表示される。
- 最近の明細に `成城石井` が表示される。

## 安定化メモ

- 金額の厳密値はAPIテスト側に寄せ、E2Eでは画面領域と代表データの表示を確認する。
- 当月判定はサンプルデータの固定日付に依存する。
