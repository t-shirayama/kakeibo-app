# アップロードE2Eシナリオ

## 対応テストファイル

- `frontend/e2e/upload.spec.ts`

## 目的

PDFアップロード画面と、完了/失敗を含む取込履歴が表示されることを確認する。

## 前提データ

- 認証済みstorage stateを使う。
- サンプルアップロード履歴として `2026_04_楽天カード.pdf` と `2026_05_読み取り不可.pdf` が含まれる。

## 操作手順

- `/upload` を開く。
- アップロード画面の見出しを確認する。
- 取り込み履歴の見出しを確認する。
- 完了済み行と失敗行を確認する。

## 期待結果

- `アップロード`、`PDF明細をアップロード`、`取り込み履歴` が表示される。
- `2026_04_楽天カード.pdf` の行に `完了` と `8件` が表示される。
- `2026_05_読み取り不可.pdf` の行に `失敗` と失敗理由が表示される。

## 安定化メモ

- 実ファイルアップロードはPDF解析fixture/APIテスト側に寄せ、E2Eでは履歴表示を確認する。
- 行単位でテキストを確認し、同名テキストの重複に依存しない。
