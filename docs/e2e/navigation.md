# ナビゲーションE2Eシナリオ

## 対応テストファイル

- `frontend/e2e/navigation.spec.ts`

## 目的

サイドナビから主要画面へ遷移でき、現在ページが強調され、設定リンクがデスクトップのサイドバー下部に配置されることを確認する。

## 前提データ

- 認証済みstorage stateを使う。
- サンプルデータの内容には依存しない。

## 操作手順

- `/dashboard` を開く。
- サイドナビのリンクを順番にクリックする。
- 対象画面はダッシュボード、明細一覧、収入設定、アップロード、カテゴリ管理、レポート、設定。
- デスクトップ幅で設定リンクの配置を確認する。

## 期待結果

- クリックごとにURLが対象パスで終わる。
- 各画面の主要見出しが表示される。
- 現在ページのリンクに `aria-current="page"` が付与される。
- 設定リンクはレポートより下にあり、サイドバー下部に配置される。

## 安定化メモ

- リンク名と見出し名を同じラベルで確認する。
- テストデータ変更による影響を避けるため、一覧の中身は検証しない。
