# 明細一覧E2Eシナリオ

## 対応テストファイル

- `frontend/e2e/transactions.spec.ts`

## 目的

明細一覧の期間絞り込み、カテゴリ絞り込み、検索、フィルタチップ、ソート、件数表示、未分類表示、キーボードフォーカス時ハイライト、手動追加、編集、同じ店名のカテゴリ一括更新確認、削除、Excelエクスポートを確認する。

## 前提データ

- 認証済みstorage stateを使う。
- サンプルデータには `成城石井`、`Amazon.co.jp`、`名称未確定の取引` が含まれる。
- `名称未確定の取引` は未分類確認用の明細として扱う。
- 手動追加する明細名は `E2Eテスト店舗` とする。
- 同じ店名の一括更新確認用に `E2E一括店舗` と `E2E単独店舗` を追加する。

## 操作手順

- `/transactions` を開く。
- 初期表示で当月明細を確認する。
- 開始日・終了日を先月へ切り替える。
- 開始日・終了日を今年へ切り替え、カテゴリを日用品に絞り込む。
- 検索結果の行へフォーカスを移す。
- 検索欄に `Amazon` を入力する。
- 金額ソートを昇順・降順に切り替える。
- フィルタ解除後、カテゴリを未分類に絞り込む。
- 検索欄に `未分類` を入力する。
- 明細を手動追加する。
- 追加した明細を編集する。
- Excelエクスポートボタンを押す。
- 編集済み明細を削除する。
- 同じ店名の明細を2件追加し、カテゴリ変更時に一括更新する。
- 同じ店名の明細を2件追加し、カテゴリ変更時に更新をキャンセルする。

## 期待結果

- 初期表示では `成城石井` が表示され、`Amazon.co.jp` は表示されない。
- 初期表示では適用中フィルタに `今月` が表示され、件数と `ソート: 日付 降順` が表示される。
- 検索対象として店名、メモ、カテゴリが明示される。
- 先月では `Amazon.co.jp` が表示され、`成城石井` は表示されない。
- 日用品カテゴリ絞り込みで `Amazon.co.jp` が表示される。
- 適用中フィルタチップに日付範囲、カテゴリ、検索語が表示される。
- 検索候補に入力中の候補が表示される。
- 検索結果の行へキーボードフォーカスしたとき、対象行がハイライトされる。
- 金額ソートの状態が昇順・降順で切り替わる。
- 金額は支出色で表示される。
- 未分類絞り込みで `名称未確定の取引` が表示される。
- `未分類` 検索でも `名称未確定の取引` が表示される。
- 手動追加、編集、削除が画面上に反映される。
- 一括更新を選ぶと、同じ店名の明細カテゴリがまとめて更新される。
- キャンセルを選ぶと、編集対象を含む同じ店名の明細カテゴリは更新されない。
- ExcelエクスポートAPIが200を返し、検索語と期間を含む現在の条件付きで呼ばれる。

## 安定化メモ

- 作成したE2E明細は同じテスト内で削除する。
- 明細フォームの保存分岐や同じ店名の一括更新確認の細かい分岐は Frontend Integration Test を主とし、E2Eでは代表的な追加・編集・削除と一括更新導線を確認する。
- Excelはファイル内容ではなくAPIの200応答を確認する。
- 期間やカテゴリの切り替え後は、対象セルの表示/非表示でAPI反映を待つ。
