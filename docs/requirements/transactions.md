# 明細一覧要件

## 対応デザイン

- `docs/designs/transactions.png`

## 目的

登録済みの明細を検索、確認、編集、削除できるようにする。

## 表示項目

- 日付
- 店名
- カテゴリ
- 金額
- 支払い方法
- メモ
- 編集操作
- 削除操作
- ページネーション

## 操作

- 店名、メモ、カテゴリでキーワード検索できる。
- 日付は開始日・終了日のカレンダー入力で期間指定できる。
- ダッシュボードなど他画面から `date_from` / `date_to` を指定して遷移した場合は、指定期間として絞り込みできる。
- カテゴリで絞り込みできる。
- 検索、日付、カテゴリの変更は適用ボタンなしで即時反映する。
- 適用中の検索条件、日付、カテゴリはチップで表示する。
- フィルタ解除ボタンで検索条件、日付、カテゴリをまとめて解除できる。
- 日付と金額で昇順・降順にソートでき、現在のソート状態を表示する。
- 検索語、期間、カテゴリ、ページ番号、ページサイズ、ソート状態はURLクエリへ同期し、再読み込み後も復元できる。
- 明細を手動追加できる。
- 明細を編集できる。
- 明細編集でカテゴリを変更する場合、同じ店名の他明細があれば一括更新するか、更新をキャンセルするか確認できる。
- 明細を削除できる。
- 明細データをExcel（`.xlsx`）でエクスポートできる。
- ページを切り替えられる。

## 業務ルール

- 一覧は日付の降順を初期表示とする。
- 同一日付の明細は登録日時の降順で表示する。
- 金額は日本円で表示する。
- 金額は右寄せ、太字、桁を読み取りやすい数字表示で表示する。
- 支出金額は赤、収入金額は緑で表示する。
- 検索対象は店名、メモ、カテゴリとし、画面上にも明示する。
- 検索入力中は店名、メモ、カテゴリから候補を提示する。
- 一覧には現在の条件に一致した件数を表示する。
- 検索結果の各行はキーボードフォーカス時にハイライトし、現在位置を視認しやすくする。
- 検索結果のスクロールはテーブル本体だけで行い、スクロール中もヘッダ行を固定表示して列の意味を見失わないようにする。
- カテゴリが未設定、または紐づくカテゴリが無効化・論理削除されている明細は、表示と未分類絞り込みで「未分類」として扱う。
- 削除は確認後に実行する。
- 同じ店名のカテゴリ一括更新を選択した場合、編集対象以外の同じ店名の明細カテゴリも同じカテゴリへ更新する。
- 同じ店名のカテゴリ一括更新をキャンセルした場合、編集対象の明細も更新しない。
- 明細編集と明細削除は監査ログとして記録する。
- 検索条件、期間、カテゴリは同時に適用できる。
- 明細一覧のExcelエクスポートは、検索語、期間、カテゴリを含む現在の検索結果に一致するデータだけを対象にする。
- 1ページあたりの初期表示件数は10件とし、設定で10件、20件、50件を選択できるようにする。
- 設定で保存した `page_size` は明細一覧の初期表示件数として使い、URLに `page_size` がある場合はその値を優先する。
- ページネーションは `page` と `page_size` を使うoffset/page方式とする。
- エクスポート形式はExcel（`.xlsx`）とする。
- Excel出力には明細一覧、カテゴリ集計、月別集計を含める。

## 例外・空状態

- 条件に一致する明細がない場合は、空状態を表示する。
- 削除に失敗した場合は、対象行が分かるエラー表示にする。
- データ取得に失敗した場合は、再読み込み可能なエラー表示にする。
