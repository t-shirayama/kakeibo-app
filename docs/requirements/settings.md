# 設定要件

## 対応デザイン

- `docs/designs/screens/settings.png`

## 目的

ユーザーが家計簿アプリの表示、データ、基本設定を変更できるようにする。

## 表示項目

- 通貨
- タイムゾーン
- 1ページあたりの件数
- 日付の表示形式
- データエクスポート
- 全データ削除

## 操作

- 通貨を設定できる。
- タイムゾーンを設定できる。
- 一覧の表示件数を設定できる。
- 日付表示形式を設定できる。
- データをエクスポートできる。
- すべてのデータを削除できる。

## 業務ルール

- 初期通貨はJPYとする。
- 通貨はJPY固定とする。
- 初期タイムゾーンはAsia/Tokyoとする。
- 日付表示形式の初期値は`YYYY/MM/DD`とする。
- 1ページあたりの件数は `10` / `20` / `50` だけを選択できる。
- 全データ削除は確認後に実行する。
- 全データ削除時はパスワード再入力または確認文字列入力を必須とする。
- 全データ削除の対象はログインユーザー本人の明細、未分類以外のカテゴリ、アップロード履歴、収入設定、保存済みPDF原本に限定する。
- `未分類` カテゴリは取込や分類表示の基盤として使うため、全データ削除でも残す。
- 設定変更後は次回表示から反映する。

## 例外・空状態

- 設定保存に失敗した場合は、変更前の状態を維持する。
- 全データ削除に失敗した場合は、削除結果が不明にならないようにエラーを表示する。
- エクスポート対象データがない場合は、空データとして扱うか、実行不可にする。
