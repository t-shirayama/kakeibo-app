# 明細詳細（編集モーダル）要件

## 対応デザイン

- `docs/designs/transaction-edit-modal.png`

## 目的

一覧画面から明細の内容をすばやく修正できるようにする。

## 入力項目

- 日付
- 店名
- カテゴリ
- 金額
- 支払い方法
- メモ

## 操作

- 明細内容を編集できる。
- 保存できる。
- キャンセルできる。
- モーダルを閉じられる。

## 業務ルール

- 日付は必須とする。
- 店名は必須とする。
- カテゴリは必須とする。
- 金額は整数JPYとし、マイナス金額は取消明細を表す。
- 0円明細を許可する。
- 支払い方法は選択式とする。
- メモは任意とする。
- メモの最大文字数は500文字とする。
- 保存時にドメインルールを満たさない場合は更新しない。
- 保存成功後は一覧に変更内容を反映する。

## 例外・空状態

- バリデーションエラーは該当項目の近くに表示する。
- 保存に失敗した場合は、モーダルを閉じずにエラーを表示する。
- 編集対象の明細が存在しない場合は、一覧を再取得する。
