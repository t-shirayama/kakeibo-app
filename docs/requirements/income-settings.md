# 収入設定要件

## 対応デザイン

- `docs/designs/income-settings.png`

## 目的

家族ごとの毎月収入を設定し、発生日を迎えた月に収入明細を自動追加できるようにする。

## 表示項目

- 対象者名
- 毎月の金額
- 毎月の発生日
- 登録開始月
- 登録終了月
- 収入カテゴリ
- 月別変更の対象月
- 月別変更の金額
- 月別変更の発生日

## 操作

- 家族や本人など対象者ごとに収入設定を追加できる。
- 収入設定の毎月の金額、発生日、カテゴリを変更できる。
- 収入設定ごとに登録開始月と登録終了月を設定できる。
- 収入設定を削除できる。
- 対象月だけ金額と発生日を上書きできる。
- 月別変更を削除できる。

## 業務ルール

- 収入設定はログインユーザー本人のデータのみを扱う。
- 登録開始月から登録終了月までの範囲で、発生日を迎えた月の収入明細は同じ収入設定・同じ対象月で重複しないよう自動追加する。
- 登録開始月を過去月にした場合は、指定期間内で発生日を過ぎている過去月の収入明細も自動追加する。
- 発生日が月末日を超える月は、その月の末日を発生日として扱う。
- 自動追加する明細は `transaction_type=income` とし、既存の `transactions` テーブルに保存する。
- 自動追加する明細の店名は `{対象者名} 収入` とする。
- 自動追加する明細の利用者は対象者名、支払い方法は `自動登録` とする。
- 対象者ごとにカテゴリを選べるため、家族別に収入カテゴリを分けられる。
- 月別変更がある月は、基本設定ではなく月別変更の金額と発生日を使う。
- 登録終了月が未設定の場合は、終了指定なしで継続する。

## 例外・空状態

- 収入設定がない場合は空状態を表示する。
- カテゴリがない場合は収入設定を追加できない。
- 無効化または削除されたカテゴリは新規・更新時に指定できない。
- 画面上に、登録開始月を過去月にすると指定期間内の過去の発生日分も自動登録されることを説明表示する。
