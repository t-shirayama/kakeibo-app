# アップロード要件

## 対応デザイン

- `docs/designs/upload.png`

## 目的

PDF明細をアップロードし、家計簿明細として取り込めるようにする。

## 表示項目

- PDFアップロード領域
- ファイル選択ボタン
- アップロード履歴
- ファイル名
- アップロード日時
- 取込件数
- ステータス
- 操作

## 操作

- PDFファイルをドラッグ＆ドロップできる。
- ファイル選択ダイアログからPDFを選択できる。
- アップロード履歴を確認できる。
- 取込結果を確認できる。
- アップロード中の進捗を確認できる。
- 失敗したアップロードを同じファイル、または選び直したファイルで再試行できる。
- 必要に応じて取込結果をダウンロードできる。

## 業務ルール

- アップロード対象はPDFファイルのみとする。
- 初期対応PDFは楽天カード明細PDFとする。
- 最大アップロードサイズは10MBとする。
- PDF抽出ライブラリは PyMuPDF を使う。
- アップロードしたPDFから明細候補を抽出する。
- 抽出した明細は日付、店名、金額、支払い方法、メモを持つ。
- 初回取込時のカテゴリは未分類とし、ユーザーが分類する。
- 次回以降は同一項目の過去分類からルールベースで自動分類する。
- 過去分類が見つからない場合は未分類にする。
- 同一明細の重複取込を防ぐ。
- 重複判定キーは `docs/specs/pdf-import.md` に従う。
- アップロード履歴には成功、失敗、処理中の状態を記録する。
- アップロード中は対象ファイル名と進捗率を画面上で確認できるようにする。
- アップロード履歴を論理削除したとき、保存済みPDF原本も即削除する。
- アップロード失敗は監査ログとして記録する。

## 例外・空状態

- PDF以外のファイルは受け付けない。
- 解析できないPDFは失敗として履歴に残す。
- 一部の明細だけ解析できない場合は、取り込めた件数と失敗理由を確認できるようにする。
- アップロード履歴がない場合は、空状態を表示する。
