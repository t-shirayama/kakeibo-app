# 監査ログ要件

## 目的

ログインユーザー本人に紐づく重要操作を、専用画面で確認できるようにする。

## 表示項目

- 日時
- 操作種別
- 対象種別
- 実行ユーザー
- 詳細

## 操作

- 監査ログを日時、操作種別、対象種別、実行ユーザー、詳細で確認できる。
- 操作種別、対象種別、開始日、終了日で絞り込める。
- ページ送りで一覧を確認できる。

## 業務ルール

- 監査ログはログインユーザー本人に紐づく重要操作だけを表示する。
- 明細更新、明細削除、アップロード失敗などの重要操作を対象にする。
- 監査ログは参照専用とし、画面から編集や削除はできない。

## 例外・空状態

- 条件に一致する監査ログがない場合は空状態を表示する。
- 取得に失敗した場合は、再読み込みの判断ができるようにエラーを表示する。
