# ダッシュボード要件

## 対応デザイン

- `docs/designs/dashboard.png`
- `docs/designs/dashboard.svg`

## 目的

ユーザーが今月の家計状況を短時間で把握できるようにする。

## 表示項目

- 今月の支出合計
- 今月の収入合計
- 今月の収支
- 取引件数
- 前月比
- カテゴリ別支出割合
- 月別の支出推移
- 最近の取引
- 明細一覧の簡易表示

## 操作

- 表示月を切り替えられる。
- 明細の検索、期間指定、カテゴリ絞り込みができる。
- 明細一覧へ遷移できる。
- 明細を手動追加できる。
- 明細データをエクスポートできる。
- PDFアップロード画面へ遷移できる。

## 業務ルール

- 支出合計は対象月の支出明細を合算する。
- 収入合計は対象月の収入明細を合算する。
- 収支は収入合計から支出合計を差し引く。
- 取引件数は対象月の明細件数を表示する。
- 前月比は対象月と前月の同種指標を比較する。
- カテゴリ別支出割合は支出明細のみを対象にする。
- カテゴリ別支出割合は円グラフで表示し、カテゴリ名、金額、割合を凡例で確認できるようにする。
- カテゴリ別支出割合の凡例にフォーカスまたはホバーした場合、対応する円グラフの対象部分をハイライト表示する。
- 最近の取引は日付の降順で表示する。
- ダッシュボードの初期表示月は当月とする。
- 最近の取引の表示件数は5件とする。

## 例外・空状態

- 明細がない場合は、集計値を0として表示する。
- グラフに表示するデータがない場合は、空状態を表示する。
- 集計取得に失敗した場合は、再読み込み可能なエラー表示にする。
