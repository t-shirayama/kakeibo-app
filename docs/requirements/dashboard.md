# ダッシュボード要件

## 対応デザイン

- `docs/designs/ToBe/dashboard.png`

## 目的

ユーザーが対象月の家計状況と支出傾向をひとつの画面で振り返れるようにする。

## 表示項目

- 収入
- 支出
- 収支
- 貯蓄率
- 各KPIの前月比
- カテゴリ別支出割合
- 直近6ヶ月の収入・支出推移
- 今月の気づき
- カテゴリ別支出の前月比テーブル

## 操作

- 表示月を月選択カレンダーで切り替えられる。
- 表示月の前後にある矢印ボタンで、前月・翌月へ切り替えられる。
- 表示月入力は前後の矢印と入力形式で用途が分かるため、画面上の固定ラベルは表示しない。ただしスクリーンリーダー向けのラベルは保持する。
- 選択月を初期値として、明細一覧へ遷移して期間を詳細に確認できる。
- カテゴリ別支出割合のカテゴリをクリックした場合、明細一覧へ遷移し、表示月と選択カテゴリで絞り込んだ明細を表示する。
- Excel（`.xlsx`）で明細データをエクスポートできる。

## 業務ルール

- 収入、支出、収支は対象月の明細から集計する。
- 貯蓄率は `収支 / 収入 * 100` で算出し、収入が0円の場合は0%として扱う。
- 前月比は対象月と前月の同種指標を比較する。
- 前月比は、支出は減少を良い変化、収入・収支・貯蓄率は増加を良い変化として、良い変化を緑、悪い変化を赤で表示する。
- KPIの主要数値は前月比より大きく太い文字で表示し、桁を読み取りやすくする。
- カテゴリ別支出割合は支出明細のみを対象にする。
- カテゴリ別支出割合は円グラフで表示し、カテゴリ名、金額、割合を凡例で確認できるようにする。
- カテゴリ別支出割合の円グラフと凡例は割合が大きい順に表示し、色はカテゴリに設定された色を使う。
- カテゴリ別支出割合は円グラフ領域の高さを固定し、カテゴリ数が多い場合は凡例一覧をスクロール表示して、他のグラフやセクションの位置が大きく崩れないようにする。
- カテゴリ別支出割合の凡例にフォーカスまたはホバーした場合、対応する円グラフの対象部分をハイライト表示する。
- 収入・支出推移は直近6ヶ月分を表示し、各月の収入と支出を2本の棒で並べて表示する。
- 収入・支出推移には縦軸の目盛を表示し、金額規模を把握できるようにする。
- 収入・支出推移には表示期間の平均支出ラインを表示する。
- 今月の気づきは、対象月のカテゴリ構成、過去6ヶ月平均との比較、貯蓄率を元に表示する。
- カテゴリ別支出の前月比テーブルは、対象月カテゴリと前月カテゴリをカテゴリID単位で比較して表示する。
- ダッシュボードの初期表示月は当月とする。
- 表示月を切り替えた場合、KPI、カテゴリ別支出割合、収入・支出推移、前月比テーブルは選択した月を基準に再取得する。
- 従来のレポート画面が持っていたExcelエクスポート導線はダッシュボードへ統合する。

## 例外・空状態

- 明細がない場合は、集計値を0として表示する。
- グラフに表示するデータがない場合は、空状態を表示する。
- 集計取得またはエクスポートに失敗した場合は、再実行可能なエラー表示にする。
